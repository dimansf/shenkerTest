Задача - создать конвертер заявок на перевозку груза.

Исходные данные:
1. Регионы.xls - отгрузочный лист клиента, содержащий информацию о заказах (1 строка - 1 заказ)
2. BI_Base.xls - справочник соответсвий кодов "код получателя в системе Шенкер" - "код места доставки в системе Шенкер" - "код получателя в системе клиента"
3. График.xls - график перевозок по городам.
4. Города.xls - справочник "места доставки" - "город".
5. blank.xlt - шаблон заявки на перевозку груза.

Алгоритм формирования заявок:
1. Сгруппировать заказы клиента по получателям.
2. Рассчитать количество груза по каждой заявке (одна заявка - несколько заказов, отправляемых в адрес одного получателя). Заявка не может быть больше 32 паллет. Один заказ не делится на разные заявки.
3. Рассчитать плановую дату доставки по каждой заявке.
4. Заполнить бланк заявки данными и сохранить в каждую заявку в отдельный файл. В имени файла должен быть указан код получателя и город места доставки.


Список используемых полей Регионы.xls:
- "Номера заказов" - должны быть проставлены в бланк заявки в поле "Номера заказов получателя"
- "Количество коробов" - суммируется по получателю (поле Код получателя) и проставляется в бланк заявки в поле "Комментарии"
- "Вес, кг" - суммируется по получателю (поле Код получателя) и проставляется в бланк заявки в поле "Вес груза, кг"
- "Объем, куб.м" - суммируется по получателю (поле Код получателя) и проставляется в бланк заявки в поле "Объем груза, м3"
- "Объем, куб.м" - суммируется по получателю (поле Код получателя), делится на 1,15, проставляется в поле "количество паллет"
- "Код получателя" - по справочнику BI_Base.xls позволяет получить коды получателя и места доставки (Client_ID, Place_ID), которые проставляются в бланк заявки в поля "Получатель    (Указывается юридический адрес организации)" и "Место доставки    (Указывается физический адрес места передачи груза)"
- Из места доставки получаем город доставки Города.xls 

- "Дата подачи груза к перевозке"- запрашивает конвертером перед генерацией заявки. По умолчанию должен предлагаться следующий рабочий день, но оператор может указать любой день.
- "Планируемая дата доставки груза" - берется согласно графику, городу места доставки и Дате подачи груза к перевозке. Если в графике указано несколько дней доставки, то надо использовать наиболее раннию. Например: Красноярск, дата подачи - ЧТ, дата доставки - пн**,вт**,ср**. В заявке нужно использовать дату доставки пн**.

В конечном счёте программа должна работать так:
1. После запуска программы оператор в диалоге выбирает файл Регионы.xls.
2. Оператор подтверждает/редактирует дату подачи либо в поле на форме, либо в специально появившемся окне. 
3. Автоматически генерируются файлы заявок.
4. Открывается папка с сохранёнными заявками.
